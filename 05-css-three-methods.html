🎨 CSS 三种引入方式完整教程

学习如何将 CSS 样式应用到 HTML 文档中

方式1️⃣：外部样式表（External Style Sheet）

📝 什么是外部样式表？

将 CSS 代码写在单独的 .css 文件中，然后在 HTML 文件的 <head> 标签内使用 <link> 标签引入。

💻 使用方法：

第1步：创建 CSS 文件（如 style.css）

/* style.css 文件内容 */ body { background-color: #f0f0f0; font-family: Arial, sans-serif; } .header { color: #333; padding: 20px; }

第2步：在 HTML 中引入

<!DOCTYPE html> <html> <head> <!-- 使用 link 标签引入外部 CSS --> <link rel="stylesheet" href="style.css"> </head> <body> <div class="header">内容</div> </body> </html>

✅ 优点：

  • 代码分离：HTML 和 CSS 完全分开，结构清晰
  • 可复用：一个 CSS 文件可以被多个 HTML 页面使用
  • 易维护：修改样式只需改一个 CSS 文件
  • 加载快：浏览器会缓存 CSS 文件，提高性能
  • 团队协作：前端工程师可以专注写 CSS

❌ 缺点：

  • 需要额外的 HTTP 请求加载 CSS 文件
  • 如果 CSS 文件很大，首次加载可能较慢
  • 需要管理多个文件
💡 最佳使用场景：
  • ✓ 大型网站项目
  • ✓ 多页面网站（多个 HTML 页面共用样式）
  • ✓ 团队协作开发
  • ✓ 需要维护的长期项目

方式2️⃣：内部样式表（Internal Style Sheet）

📝 什么是内部样式表？

在 HTML 文件的 <head> 标签内使用 <style> 标签， 直接编写 CSS 代码。本页面就是使用这种方式！

💻 使用方法：

<!DOCTYPE html> <html> <head> <!-- 在 head 中使用 style 标签 --> <style> body { background-color: #f0f0f0; } .box { padding: 20px; border: 1px solid #ccc; } </style> </head> <body> <div class="box">内容</div> </body> </html>

📌 实际演示：

这个盒子的样式就是通过内部样式表定义的！ 查看本页面的源代码，在 <head> 中可以看到 <style> 标签。

✅ 优点：

  • 集中管理：所有样式在一个地方，容易查找
  • 无需额外文件：不需要创建单独的 CSS 文件
  • 独立性：每个页面可以有自己的样式
  • 适合小项目：简单快速，适合单页面应用
  • 减少请求：不需要额外的 HTTP 请求

❌ 缺点：

  • 样式不能在多个页面间复用
  • HTML 文件会变大
  • 不利于浏览器缓存
  • HTML 和 CSS 混在一起，不够清晰
💡 最佳使用场景：
  • ✓ 单页面应用或演示页面
  • ✓ 学习和测试 CSS
  • ✓ 某个页面有独特的样式（其他页面用不到）
  • ✓ 快速原型开发

方式3️⃣：内联样式（Inline Style）

📝 什么是内联样式？

直接在 HTML 标签上使用 style 属性编写 CSS。 这是优先级最高的样式方式。

💻 使用方法：

<div style="color: red; font-size: 20px; padding: 10px;"> 这是内联样式 </div> <p style="background-color: yellow; margin: 10px;"> 每个标签都可以有自己的 style 属性 </p>

📌 实际演示：

这个盒子使用了内联样式！ 所有样式都直接写在 style 属性中。

这个段落也使用了内联样式，包括渐变背景！

✅ 优点：

  • 最高优先级：会覆盖其他方式的样式
  • 快速调试：临时测试样式很方便
  • 精确控制：可以为单个元素定制样式
  • 动态生成：适合用 JavaScript 动态添加样式

❌ 缺点：

  • 难以维护：样式分散在各个标签中
  • 不可复用：每个元素都要单独写
  • 代码混乱：HTML 代码会变得很长
  • 效率低：修改样式需要逐个修改标签
  • 不利于 SEO：增加 HTML 文件大小
💡 最佳使用场景：
  • ✓ 临时测试和调试
  • ✓ 动态生成样式（JavaScript 控制）
  • ✓ 覆盖其他样式（紧急修复）
  • ✓ Email HTML 模板（兼容性要求）
  • ⚠️ 日常开发中尽量少用！

🎯 CSS 样式优先级（重要！）

📌 优先级规则（从高到低）：

1️⃣ 内联样式 (style 属性)
↓
2️⃣ 内部样式表 (<style> 标签)
↓
3️⃣ 外部样式表 (.css 文件)
↓
4️⃣ 浏览器默认样式

💡 特殊规则：
  • !important 可以提升优先级（但不推荐滥用）
  • 相同优先级时，后面的样式会覆盖前面的
  • ID 选择器 > 类选择器 > 标签选择器

🏆 最佳实践建议

✅ 推荐的做法：

  1. 大型项目：使用外部样式表
    • 创建独立的 CSS 文件（如 style.css）
    • 可以进一步分模块（header.css, footer.css）
    • 利用浏览器缓存提高性能
  2. 单页面或小项目：使用内部样式表
    • 适合学习和快速原型
    • 所有代码在一个文件，便于分享
  3. 特殊情况：谨慎使用内联样式
    • 仅用于临时调试
    • JavaScript 动态控制样式
    • Email HTML 模板

❌ 不推荐的做法：

  • 在生产环境大量使用内联样式
  • 所有项目都用内部样式表（不利于维护）
  • 滥用 !important
  • 三种方式混乱使用，没有统一规范

🌟 实际项目中的使用建议

典型的项目结构：

/* 项目文件结构 */ project/ ├── index.html <-- HTML 文件 ├── css/ │ ├── style.css <-- 主样式文件（外部样式表） │ ├── header.css <-- 头部样式 │ └── footer.css <-- 底部样式 ├── js/ │ └── script.js └── images/

在 HTML 中引入：

<head> <!-- 推荐：使用外部样式表 --> <link rel="stylesheet" href="css/style.css"> <link rel="stylesheet" href="css/header.css"> <!-- 特殊情况：某些页面独有的样式可以用内部样式表 --> <style> .special-page-element { color: blue; } </style> </head>

📚 学习总结

1. 三种引入方式：

  • ✅ 外部样式表（.css 文件）- 最推荐
  • ✅ 内部样式表（<style> 标签）- 适合小项目
  • ⚠️ 内联样式（style 属性）- 谨慎使用

2. 优先级顺序：

内联样式 > 内部样式表 > 外部样式表 > 浏览器默认

3. 选择建议：

  • 📁 多页面网站 → 外部样式表
  • 📄 单页面应用 → 内部样式表
  • 🔧 临时调试 → 内联样式
